# UI 표시 보정 작업 큐

이 문서는 `docs/pingpong.txt`의 후속 인계 보조 파일이다.
현재 `docs/pingpong.txt` 직접 갱신이 실패할 경우, 이 파일을 먼저 읽고 작업한다.

## 작업 해석 기준

- `수정해`, `고쳐`, `반영해`, `넣어`, `정리해`, `진행해`는 실행 지시다.
- `검토만`, `수정하지 마`, `핑퐁 X`, `커밋하지 마`, `아직 하지 마`가 있을 때만 쓰기 작업을 멈춘다.
- 쓰기 전 최신 파일 SHA를 확인한다.
- 409가 나면 덮어쓰지 말고 최신 파일을 다시 읽는다.
- 작업 후 변경 파일, 검증, 빌드 여부를 보고한다.

## 1순위 작업 묶음: UI 표시 보정

본문 지연 로딩보다 먼저 처리한다.
새 콘텐츠 확장이 아니라 현재 플레이 화면의 의미 오류를 바로잡는 작업이다.

### 1. DialogueCard 텍스트 계층 분리

- 상단 턴 정보는 유지한다.
- 카드 내부는 `현재 턴의 제목 -> 구분선 -> 화자 -> 대사` 순서로 분리한다.
- `DialogueCard.jsx`와 CSS 중심으로 수정한다.
- `ChoicePanel` 전체를 뜯지 않는다.
- 선택지 목록 UI와 첫날 힌트 팝업은 이번 수정에 끌어들이지 않는다.

### 2. 꿈/회차 표기 보정

- 현재 `dream-mark`가 날짜와 묶여 있다.
- 꿈 번호는 `day`가 아니라 실제 회차/기록 축에서 가져온다.
- `1번째 꿈` 문구는 없앤다.
- `첫 번째 꿈`, `두 번째 꿈`처럼 자연스러운 표기를 우선 검토한다.
- 밤 표기는 날짜 기준으로 유지 가능하다.

### 3. 성향 레벨 음수 변화 보정

- 성향 레벨이 높을수록 보너스뿐 아니라 음수 변화도 커지는 것으로 보인다.
- 성향 레벨 보정은 긍정 효과에 우선 적용한다.
- 음수 delta에 같은 배율을 적용하는 구조는 재검토한다.
- verify에 양수/음수 delta 배율 차이를 검사한다.

### 4. 역방향 수치 색상 보정

- 일부 수치는 증가할수록 좋지 않은 변화다.
- 해당 값의 증가는 danger 계열 색으로 표시한다.
- 해당 값의 감소는 완화색으로 표시한다.
- 단순히 delta 부호만으로 positive/negative class를 정하지 않는다.

### 5. 확정 선택 표시 보정

- 일부 선택은 확률 판정이 아니라 확정 선택이어야 한다.
- 해당 선택은 표시와 처리 모두 고정값으로 둔다.
- 관련 보정값이 이 표시를 바꾸지 않는다.
- verify에 고정값 검사를 추가한다.

### 6. 진행 카운터 표시 보정

- 카운터가 은폐 표시로 시작한다.
- 0일 때는 은폐 표시를 유지할 수 있다.
- 1 이상부터는 의미가 드러나야 한다.
- 저장/로드 후에도 표시가 유지되어야 한다.
- 관련 선택 후 카운터가 즉시 바뀐다.

## 검증

- `npm.cmd run verify`
- `npm.cmd run build`
- 기존 verify를 약화하지 않는다.
- 자동 플레이 smoke 검증을 약화하지 않는다.
- 작업 후 `docs/work_log.txt`에 기록한다.
