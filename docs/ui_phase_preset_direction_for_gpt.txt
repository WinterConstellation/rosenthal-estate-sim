# UI 단계 프리셋 방향 메모

목적: 낮/밤 UI를 평화 상태와 이상현상 단계로 분리하려는 방향을 다른 GPT/코덱스가 이해할 수 있게 정리한다.

이 문서는 당장 구현 지시가 아니다.
현재 의견을 종합한 판단 메모이며, 실제 구현 전에는 기존 코드 구조와 연출 비용을 확인해야 한다.

## 핵심 판단

UI는 시간대와 이상현상 단계를 분리해서 보는 편이 좋다.

시간대는 `day`와 `night`로 나누고, 이상현상 단계는 최대 3단계로 제한한다.

- `calm`: 평화로움, 일상, 기본 상태
- `anomaly`: 최초 이상현상 발생, 공포 UI 일부 출현
- `critical`: 임계치 근접, UI 붕괴/오염 효과 강화

이렇게 하면 총 6개 프리셋으로 정리할 수 있다.

- `day-calm`
- `day-anomaly`
- `day-critical`
- `night-calm`
- `night-anomaly`
- `night-critical`

핵심은 기존 붉은 밤 배경을 기본 밤으로 쓰지 않는 것이다.
기본 밤은 푸르스름하고 예쁜 일상 밤이어야 하고, 붉은 밤은 이상현상 이후의 상태로 격상하는 편이 효과적이다.

## 낮 프리셋

### day-calm

기존 낮 화면에 해당한다.
평화로운 영지 관리와 일상 흐름이 중심이다.

- 따뜻한 낮 배경
- 기존 빛무리/먼지/햇빛 반사 계열 글리프 효과 유지
- 공포 UI 표시 없음
- 공포도, 이상현상, 침식 수치는 숨김
- 빨간 선택지는 사용하지 않거나 극히 제한

이 단계의 글자 효과는 공포 텍스트가 아니라 낮의 분위기 효과로 취급한다.

### day-anomaly

낮에 최초 이상현상이 발생한 상태다.
공포 UI가 처음 모습을 드러낼 수 있지만, 아직 과하게 붉거나 망가진 연출은 아껴 쓰는 편이 좋다.

- 낮 배경은 유지
- 짧은 분위기 문장 또는 흐릿한 경고 표시
- 공포 UI 일부 출현
- 선택지 색상은 대부분 유지
- 빨간 선택지는 정말 특별한 경우에만 사용

이 단계는 유저에게 "낮도 완전히 안전하지 않다"는 사실을 처음 알려주는 역할이다.

### day-critical

낮인데도 임계치에 가까워진 상태다.
정상적인 낮 화면이 일부 망가지기 시작한다.

- UI 일부가 깨지거나 흔들리는 효과
- Canvas 글자 효과를 이용한 오염/침식 표현
- 텍스트 일부 누락, 번짐, 미세한 위치 흔들림 가능
- 공포/이상현상 UI가 제한적으로 또는 직접적으로 표시
- 낮의 평화로운 색감이 부분적으로 변질

이 단계는 밤 연출을 낮에 그대로 복사하는 것이 아니라, 낮의 정상성이 무너지는 느낌으로 가는 편이 좋다.

## 밤 프리셋

### night-calm

새로 만들어야 하는 기본 밤이다.
푸르스름한 배경의 흔하지만 예쁜 디자인을 목표로 한다.

- 푸른빛 또는 남색 계열 배경
- 일상적인 밤, 조용한 저택 분위기
- 낮의 빛무리 효과를 색상만 푸른빛으로 변경한 글리프 효과 사용 가능
- 공포 UI 표시 없음
- 붉은색은 최대한 아낌

이 단계가 있어야 이후 붉은 밤으로 변하는 순간이 의미를 갖는다.
처음부터 밤이 붉으면 붉은색이 위험 신호가 아니라 기본 배경처럼 보일 수 있다.

### night-anomaly

기존 붉은 밤 배경은 이 단계에 배치하는 편이 좋다.
최초 이상현상 이후의 밤이며, 공포 UI가 본격적으로 출현한다.

- 기존 붉은 밤 배경 사용 후보
- 공포 UI 출현
- 붉은 텍스트 비 또는 불길한 글리프 효과 사용 가능
- 선택지와 이벤트에서 위험 신호 강화
- 공포/이상현상 수치 표시 가능

이 단계부터 유저는 밤이 평범한 시간이 아니라는 것을 명확하게 인지한다.

### night-critical

기존 붉은 밤 배경에서 추가 붕괴/침식 효과가 붙는 단계다.
구체적인 효과는 별도 안내 후 설계한다.

- 기존 붉은 밤 배경 기반
- UI 일부 붕괴, 오염, 왜곡 강화
- Canvas 기반 글자 효과 추가 가능
- 공포 수치, 이상현상, 침식 상태가 화면에 직접 영향을 줌
- 눈동자, 붉은 문장, 글자 비, 텍스트 소실 효과 같은 고강도 연출 후보

아직 세부 구현은 미정이다.
사용자가 별도 안내할 때까지 이 단계의 구체 효과를 임의로 확정하지 않는다.

## 이름과 코드 방향

기본 낮/기본 밤의 글자 효과를 `horror`나 `rain`으로만 부르면 오해가 생길 수 있다.
평화 상태의 글리프 효과와 공포 상태의 글리프 효과를 이름부터 분리하는 편이 안전하다.

후보 이름:

- `AmbientGlyphCanvas`: 평화 상태와 일반 분위기 글리프 담당
- `day-glow`: 낮의 따뜻한 빛무리
- `night-blue-glow`: 평화로운 밤의 푸른 빛무리
- `night-red-rain`: 이상현상 이후 붉은 밤 글자 효과
- `critical-corruption`: 임계치 근접 오염/붕괴 효과

기존 `HorrorTextRain`, `textRain`, `rainGlyphs` 같은 이름은 낮 기본 상태에 쓰이면 오독될 수 있다.
낮의 글자 효과는 공포 UI가 아니라 분위기 UI라는 점을 코드명과 주석에서 구분하는 편이 좋다.

## 다른 의견 메모와의 종합

`docs/intro_horror_ui_direction_for_gpt.txt`의 의견과 충돌하지 않는다.
그 문서에서는 낮의 떠다니는 글자 계열 연출을 공포 UI가 아니라 빛무리/먼지/햇빛 반사에 가까운 의도로 봤다.
이번 6프리셋 구조에서는 그 의견을 `day-calm`과 `night-calm`에 반영한다.

즉, 같은 글리프 계열이라도 상태에 따라 의미가 달라진다.

- `day-calm`: 햇빛, 먼지, 평화로운 빛무리
- `night-calm`: 달빛, 푸른 밤, 조용한 빛무리
- `night-anomaly`: 붉은 밤, 공포 텍스트, 위험 신호
- `critical`: 오염, 붕괴, 화면 침식

## 구현 전 주의

- 이 문서는 당장 만들라는 지시가 아니다.
- 3단계 이상으로 세분화하지 않는다.
- 기본 밤은 새로 만들 필요가 있다.
- 기존 붉은 밤 배경은 `night-anomaly` 후보로 본다.
- `night-critical`의 추가 효과는 사용자가 따로 안내할 때까지 확정하지 않는다.
- 공포 수치를 숨기는 것은 계산을 멈추는 뜻이 아니다. 내부 상태는 계속 유지한다.
- 빨간색과 UI 붕괴 효과는 초반에 과하게 쓰지 않는다.

## 결론

6프리셋 구조가 현재 방향에 가장 적합해 보인다.

- 기본 낮과 기본 밤은 예쁘고 안정적인 일상 상태로 둔다.
- 이상현상 이후에만 공포 UI와 붉은 배경을 본격적으로 켠다.
- 임계치 근접 단계에서는 UI 붕괴와 Canvas 글자 효과를 강화한다.
- 이름부터 `ambient`와 `horror/corruption`을 분리해 후속 작업자가 오해하지 않게 한다.
