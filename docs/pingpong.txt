# 핑퐁 파일 정리 및 실제 수정 내역

사용자의 요청에 의해 이 파일의 이전 핑퐁 내용은 수시로 삭제한다.
이 문서는 실제 코드 수정 내역과 다음 작업자가 알아야 할 판단만 남긴다.

## 최신 작업 보고

기준 시각: 2026-06-18 17:39 KST

### 반영된 커밋

- `6b539d7` `Split UI phase presets`
  - 낮/밤 UI를 `calm` / `anomaly` / `critical` 단계로 분리했다.
  - 기본 밤은 푸른 `night-calm` 배경으로 이동했다.
  - 기존 붉은 밤은 `night-anomaly` / `night-critical` 계열로 이동했다.
- `69f7d70` `Normalize line endings`
  - 이전 커밋에서 섞인 줄바꿈을 LF로 정규화했다.
  - 기능 변경은 없다.
- 이번 작업
  - 개발자툴에 `UI preset` 미리보기 선택을 추가했다.
  - 개발자툴에 `glyph format` 강제 선택을 추가했다.
  - 두 기능 모두 저장 데이터는 바꾸지 않는 화면 표시 override다.
- 이번 작업 추가
  - 개발자툴에 `Horror State Preset` 버튼을 추가했다.
  - 공포/오염/진실/변질 상태를 빠르게 만들고 기존 파생 공포 계산을 다시 태운다.
  - `reset visible horror`는 공포 수치와 개발자 preview/강제 눈 표시를 함께 낮춘다.

## 현재 구현 상태

### `src/App.jsx`

- `resolveUiPresentation(game, effectiveIsNight, horrorDirector)`를 추가했다.
- 반환 구조는 다음 기준이다.
  - `timePhase`: `day` / `night`
  - `uiStage`: `calm` / `anomaly` / `critical`
  - `preset`: `day-calm`, `day-anomaly`, `day-critical`, `night-calm`, `night-anomaly`, `night-critical`
  - `glyphFormat`: 현재 UI 단계에 맞는 글리프 서식
  - `allowHorrorLayers`: calm 단계에서는 공포 레이어를 숨기기 위한 플래그
- `applyUiPresentationToHorrorDirector`를 추가했다.
  - `glyphAtmosphere`는 calm 단계에서도 유지한다.
  - `textMist`, `staticRows`, `textEyes`는 `allowHorrorLayers`가 true일 때만 켠다.
- 이상현상 판정 기준은 현재 임시값이다.
  - `corruption >= 22`
  - `fear >= 28`
  - `intrusionPressure >= 18`
  - `monsterization >= 18`
  - `truthDiscovered`
  - `route === "altered"`
  - 변이 동료 존재
- 임계치 판정 기준도 현재 임시값이다.
  - `corruption >= 58`
  - `fear >= 65`
  - `intrusionPressure >= 55`
  - `monsterization >= 55`
  - 눈동자 강도 `>= 0.68`
  - `escape-transformed-choice`
  - `record-stop`
- 밤이라는 이유만으로 `anomaly`가 되지는 않게 했다.
- `SceneImage`는 `preset`을 받아 장면 class를 추가한다.
- `night-calm`에서는 장면 문구가 푸른 밤 분위기에 맞게 바뀐다.
- 개발자툴 전용 `applyDeveloperUiOverrides`를 추가했다.
  - `UI preset`이 `auto`가 아니면 `day/night`와 `calm/anomaly/critical` 프리셋을 로컬 state로 강제한다.
  - `glyph format`이 `auto`가 아니면 글리프 서식만 별도로 강제한다.
  - `critical` 프리셋 미리보기에서는 저장값을 바꾸지 않고 렌더링 직전 director에만 최소 강도를 보강해 static row와 눈 레이어를 확인할 수 있게 한다.
  - 예전에 제거한 떠다니는 문장 레이어(`textMist`)는 강제로 되살리지 않는다.
  - 실제 저장 데이터, 진행도, 이벤트 스크립트는 바꾸지 않는다.
- 기존 `force night preview`는 남겨두되, `UI preset`이 선택되면 그 값이 우선한다.
- 개발자툴 전용 `applyDeveloperHorrorPresetToGame`을 추가했다.
  - 버튼은 현재 회차의 개발자 테스트용 state만 바꾼다.
  - `syncDeveloperHorrorState`를 재사용해 `derivedHorror`, `revealedHorrorTraits`, `revealedHorrorStates`를 다시 계산한다.
  - `critical` 프리셋은 첫 생존 동행자가 있으면 테스트용으로 `transformed` 상태로 바꿔 눈/임계 UI를 확인하기 쉽게 한다.
  - `reset visible horror`는 동행자 변질 상태도 생존 상태로 되돌리고, 개발자 preview select와 eye override를 끈다.

### `src/styles.css`

- `.theme-night`는 밤 공통 변수만 담당하게 조정했다.
- `.theme-night-calm`을 추가했다.
  - 푸르스름한 밤 배경
  - 푸른 빛무리 계열 accent
  - 붉은 공포 레이어 없음
- `.theme-night-anomaly`와 `.theme-night-critical`을 추가했다.
  - 기존 붉은 밤 배경은 여기로 이동했다.
- `.estate-scene--night-calm` 계열은 푸른 shade/filter를 사용한다.
- `.estate-scene--night-anomaly` / `.estate-scene--night-critical`은 기존 붉은 shade/filter를 사용한다.
- `.theme-night-calm .glyph-atmosphere-canvas`는 낮보다 어둡고 푸른 분위기로 보이게 opacity만 조정했다.

## 글리프 분위기 서식 현황

- `day-glow`: 낮의 따뜻한 빛무리
- `night-blue-glow`: 평화로운 밤의 푸른 빛무리
- `night-red-rain`: 이상현상 밤의 붉은 글자비
- `critical-corruption`: 임계치 근접/오염 계열 글리프

## 완료된 것

- 기본 밤 `night-calm` 생성
- 기본 밤 푸른 배경 적용
- 기본 밤에서 공포 UI 숨김
- 기존 붉은 밤을 `night-anomaly` / `night-critical` 쪽으로 이동
- UI 단계 director 추가
- `glyphFormat`을 UI 단계에 맞게 연결
- 줄바꿈 LF 정규화
- 개발자툴 `UI preset` 미리보기 추가
- 개발자툴 `glyph format` 강제 선택 추가
- 개발자툴 `Horror State Preset` 버튼 추가

## 아직 하지 않은 것

- `night-critical`의 구체적인 전용 연출은 아직 만들지 않았다.
- 밸런스, 저장 구조, 선택지 데이터, 이벤트 스크립트는 건드리지 않았다.
- 로컬 빌드/실행 검증은 하지 않았다. 사용자가 로컬에서 직접 확인하기로 했다.
- 낮 `day-anomaly` / `day-critical`의 상세 시각 효과는 최소 연결만 되어 있고, 별도 디자인은 아직 없다.

## 개발자툴 확장 계획

목표: 개발자툴을 단순 수치 편집기가 아니라 화면 테스트 조종석으로 확장한다.
지나친 방어적 코딩보다 구현도를 우선하고, 사용자가 직접 눈으로 확인할 수 있는 화면 변화를 먼저 만든다.

### 1차 구현: UI Preview 섹션

구현 완료.

- 선택값: `auto`, `day-calm`, `day-anomaly`, `day-critical`, `night-calm`, `night-anomaly`, `night-critical`
- 실제 저장 데이터는 바꾸지 않는다.
- 로컬 state로만 화면 표시를 override한다.
- 기존 `force night preview`보다 상위 개념으로 본다.
- 이 기능으로 배경, theme class, 공포 레이어 표시 여부, 글리프 서식이 한 번에 확인되어야 한다.

### 2차 구현: Glyph Format Override

구현 완료. UI 프리셋과 별개로 글리프 서식만 강제 확인하는 기능이다.

- 선택값: `auto`, `day-glow`, `night-blue-glow`, `night-red-rain`, `critical-corruption`
- 실제 저장 데이터는 바꾸지 않는다.
- 사용자가 랙, 눈 피로, 밀도, 분위기를 직접 확인하기 위한 기능이다.
- 성능 판단은 사용자가 로컬에서 직접 테스트한다.

### 3차 구현: Horror State Preset 버튼

구현 완료. 숫자를 하나씩 입력하지 않고 장면 상태를 빠르게 만들기 위한 버튼이다.

후보:

- `calm baseline`
- `mild anomaly`
- `high fear`
- `high corruption`
- `truth discovered`
- `altered route`
- `critical`
- `reset visible horror`

주의:

- 기존 `syncDeveloperHorrorState` 흐름을 재사용한다.
- 과한 예외 처리보다 실제 테스트 가능한 상태 전환을 우선한다.

### 4차 구현: 제한적 Phase Jump

처음부터 모든 phase를 지원하지 않는다.
부속 데이터가 적고 안전한 phase부터 추가한다.

1차 후보:

- `day`
- `nightfall-transition`
- `day-eight`
- `record-stop`
- `ending`

보류:

- `night-companion`
- `night-direction`
- `expedition`
- `finale`

보류 이유: 탐사/동행자/진행도 등 부속 데이터가 필요해서 잘못 점프하면 화면만 깨질 수 있다.

### 5차 구현: 빠른 상태 조작

8일차 분기와 결산 확인용이다.

- `day`
- `dayTurn`
- `route`: `normal` / `altered` / `none`
- `sacrificeCount`
- `truthDiscovered`

### 이후 후보

- companion 상태 편집: `alive`, `dead`, `missing`, `transformed`
- debug snapshot 복사 버튼
- 낮 `day-anomaly` / `day-critical` 전용 표현 조정
- `night-critical` 전용 연출

## 주의 사항

- 이전 PowerShell CPU 문제는 원격 파일 내용 문제가 아니라, 커밋 생성용 로컬 임시 스크립트가 큰 파일을 JSON 직렬화하려다 발생한 문제다.
- 해당 임시 스크립트는 레포에 올라가지 않았다.
- 이후 큰 파일 원격 커밋 시 PowerShell `ConvertTo-Json`으로 파일 전체를 포장하지 말 것.
- 로컬 테스트/빌드/검증은 사용자가 직접 진행한다.
- 개발자툴 확장은 저장 구조와 엔진 규칙을 건드리지 않는 선에서 먼저 진행한다.
