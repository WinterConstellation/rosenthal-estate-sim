# 핑퐁 파일 정리 및 실제 수정 이력

이 문서는 최신 작업 인계용이다.
길어지면 이전 논의와 폐기안을 지우고, 다음 작업자가 바로 이어받을 현재 판단과 확인 항목만 남긴다.

## 최신 기준

기준 원격: `origin/main`
기준 시각: 2026-06-19 19:04 KST

## 현재 확정 상태

- 성흔/낙인 표식은 총 `100개`가 정본이다.
  - 성흔 50개
  - 낙인 50개
  - 성향형 94개
  - 단독 6개
- `MARK_BRANCH_UNLOCKS`는 현재 총량 안에서 모두 도달 가능하다.
- 기존 저장 마이그레이션(`LEGACY_STIGMA_MARK_MAP`)은 유지한다.
- UI 단계는 `day-calm`, `day-anomaly`, `day-critical`, `night-calm`, `night-anomaly`, `night-critical` 6단계다.
- 기본 밤은 푸른 `night-calm`이며 붉은 밤은 `night-anomaly` 이후에 사용한다.

## 이번 작업 의도

- 렉 원인이 될 수 있는 글리프 분위기 Canvas를 완전히 제거한다.
- 표식 효과와 저장 구조는 바꾸지 않고 도감 문장과 획득 방향만 보강한다.
- 이미 존재하는 분기 해방 키를 플레이어가 수집 진행도로 확인할 수 있게 한다.

## 완성된 구조

- 글리프 분위기 효과:
  - `src/effects/GlyphAtmosphereCanvas.jsx`와 `src/effects/glyphAtmosphere.css`를 삭제했다.
  - `src/App.jsx`의 import, director 상태, `glyphFormat`, Canvas 렌더링 경로를 제거했다.
  - 공포 오버레이는 글자 안개, 정적 문장, 텍스트 눈동자만 사용한다.
- 표식 도감:
  - 모든 표식은 정적 데이터인 `codexText`, `sourceHint`를 가진다.
  - 두 필드는 효과 계산과 저장 데이터에 관여하지 않는다.
  - 보유 표식은 도감 문장, 미획득 표식은 획득 힌트, 미해금 표식은 수집 조건을 표시한다.
- 분기 진행도:
  - `getMarkBranchProgress(markIds)`는 요구 조건별 현재값, 요구값, 남은 수량, 해방 여부를 반환한다.
  - `getMarkBranchLabel(branchKey)`는 수량 분기와 성향별 capstone 분기에 사용자용 이름을 제공한다.
  - 장비/도감 화면은 해방된 분기와 성흔·낙인·혼합 축의 다음 목표를 최대 1개씩 표시한다.

## 검증 기준

- `scripts/verify-game.mjs`가 다음을 검사한다.
  - `MARKS=100`, 성흔 50, 낙인 50, 분기 조건 도달 가능성
  - 모든 표식의 `codexText`, `sourceHint`
  - 분기 진행도의 0개, 일부 수집, 전체 수집 결과
  - 삭제된 글리프 파일과 `App.jsx` 글리프 참조 부재
- 로컬 실행은 사용자가 진행하므로 이번 작업에서 `npm.cmd run verify`, `npm.cmd run build`는 실행하지 않는다.
- 로컬 확인 항목:
  - 장비/도감에서 보유·미획득·미해금 문구가 구분되는지
  - 다음 해방 목표가 수집량에 따라 갱신되는지
  - 글리프 없이 글자 안개·정적 문장·눈동자가 기존 조건에서 동작하는지

## 파일 작성 규칙

- 텍스트 파일은 UTF-8(BOM 없음)·LF를 유지한다.
- Windows PowerShell의 `>`, `>>`, `Out-File`로 소스/문서를 작성하지 않는다.
- 수정은 패치 또는 UTF-8을 명시하는 도구를 사용하며, 전수 인코딩 검사는 이상 징후가 있을 때만 수행한다.
