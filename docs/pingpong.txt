# 핑퐁 파일 정리 및 실제 수정 이력

이 문서는 최신 작업 인계용이다.
길어지면 이전 논의와 중간 자기반성은 지우고, 다음 작업자가 바로 이어받을 현재 판단과 후보만 남긴다.

## 최신 기준

기준 원격: `origin/main`
기준 시각: 2026-06-19 KST

## 현재 확정 상태

- 성흔/낙인 표식은 총 `100개`가 정본이다.
  - 성흔 50개
  - 낙인 50개
  - 성향형 94개
  - 단독 6개
- `MARK_BRANCH_UNLOCKS`는 현재 총량 안에서 모두 도달 가능해야 한다.
  - 성흔 단독 축: 3 / 10 / 20 / 30 / 40 / 50
  - 낙인 단독 축: 3 / 10 / 20 / 30 / 40 / 50
  - 혼합 축: 성흔 25 + 낙인 25, 성흔 50 + 낙인 50
  - 전체 완결: total 100
- `scripts/verify-game.mjs`는 표식 총량, 성흔/낙인 50개씩, unlock 조건 도달 가능성을 검증한다.
- 기존 저장 마이그레이션(`LEGACY_STIGMA_MARK_MAP`)은 유지한다.

## UI/글리프 기준

- UI 단계는 `day-calm`, `day-anomaly`, `day-critical`, `night-calm`, `night-anomaly`, `night-critical` 6단계다.
- 기본 밤은 붉은 공포 배경이 아니라 `night-calm`의 푸른 밤/조용한 밤이다.
- 붉은 밤/오염 계열은 `night-anomaly` 또는 `night-critical` 쪽에서만 강하게 사용한다.
- 글리프 분위기 효과는 별도 효과 모듈로 격리한다.
  - 구현: `src/effects/GlyphAtmosphereCanvas.jsx`
  - 스타일: `src/effects/glyphAtmosphere.css`
  - `src/App.jsx`는 글리프 렌더러 내부 구현을 직접 소유하지 않는다.
- dev 패널에서는 글리프 형식 override를 제거했다.
  - dev는 UI 프리셋, 공포 상태 프리셋, 눈 override, 수치 편집에 집중한다.
  - 글리프 형식은 UI 프리셋과 director 상태에서 자동 결정한다.
- 이전 임시 보정 파일 `src/glyph-fall-fix.css`와 `index.html` 로드는 제거한다.

## 파일 작성 규칙

- 텍스트 파일은 UTF-8(BOM 없음)·LF를 유지한다.
- Windows PowerShell의 `>`, `>>`, `Out-File`로 소스/문서를 작성하지 않는다.
- 수정은 패치 또는 UTF-8을 명시하는 도구를 사용하며, 전수 인코딩 검사는 이상 징후가 있을 때만 수행한다.

## 확인 완료 기준

- 원격 기준 표식 상태:
  - `MARKS=100`
  - `stigma=50`
  - `brand=50`
  - 도달 불가능한 unlock 조건 없음
- 로컬 확인:
  - `npm.cmd run verify`
  - `npm.cmd run build`

## 해야 할 것 제안서

### 0. 선결: 글리프 효과 처리 방향 확정

현재 구조에서 새 요소를 추가하기 전에 글리프 효과의 운용 방향을 먼저 고정한다.
렉이 계속 체감되면 글리프 효과는 유지가 아니라 제거 대상으로 본다.

- 제거안:
  - `src/effects/GlyphAtmosphereCanvas.jsx` 삭제
  - `src/effects/glyphAtmosphere.css` 삭제
  - `src/App.jsx`의 `GlyphAtmosphereCanvas`, `createGlyphAtmosphereEffect`, `applyGlyphAtmospherePresentation`, `getGlyphAtmosphereFormat` import 및 사용 경로 삭제
  - `director.glyphAtmosphere` 생성 경로 삭제
  - `index.html`의 `src/glyph-fall-fix.css` 로드 제거
  - `src/glyph-fall-fix.css` 삭제
  - `verify-game.mjs`에 글리프 잔재 금지 검증 추가
- 유지안:
  - 글리프 구현은 반드시 `src/effects` 안에만 둔다.
  - `src/App.jsx`는 렌더러 내부 구현을 직접 소유하지 않는다.
  - dev 패널의 글리프 형식 override는 되살리지 않는다.
  - `src/glyph-fall-fix.css` 같은 임시 보정 파일은 금지한다.

### 1. 성흔/낙인 효과 밸런스 1차 정리

표식 수량은 더 늘리지 않는다.
지금은 `100개` 고정 상태에서 각 표식이 너무 같은 효과로 찍히지 않도록 효과의 맛을 나누는 단계다.

- 추가 후보:
  - `effectProfile`
  - `effectOverride`
  - `codexText`
  - `sourceHint`
- 금지:
  - 새 표식 추가
  - 성흔/낙인 일대일 대응 규칙
  - 새 자원/새 능력치/새 공포 특성
  - 획득 로직 전면 교체

### 2. 도감 설명 확장

현재 자동 설명은 기능 설명에 가깝다.
도감 수집물로 보이게 하려면 효과와 분리된 짧은 문장 필드가 필요하다.

- 권장 필드:
  - `codexText`: 도감 문장
  - `sourceHint`: 획득 방향 힌트
- 원칙:
  - 도감 문장은 효과 계산에 관여하지 않는다.
  - 본문 정본을 직접 설명하지 않는다.
  - 표식 이름의 분위기만 짧게 보강한다.

### 3. 분기 해방 체감 보강

`MARK_BRANCH_UNLOCKS`는 현재 총량 안에서 도달 가능하다.
다음 단계는 unlock key가 열린 사실을 플레이어가 작게 체감하게 하는 것이다.

- 추가 후보:
  - 규칙 탭에 해방 문구 표시
  - 특정 선택지 tooltip에 짧은 힌트 추가
  - 8일차/엔딩 진입 조건 설명에 unlock key 반영
- 금지:
  - 새 루트 자동 개방
  - 엔딩 조건 대규모 재작성
  - 해방 키를 즉시 강한 보상 버튼으로 만들기

### 4. 초반 1~3일차 흐름 정리

초반은 공포를 바로 터뜨리지 않고, 영지 경영 튜토리얼처럼 보이게 한다.
붉은색은 최대한 아끼고, 기본 밤은 푸른 `night-calm`로 둔다.

- 1일차 낮:
  - 영지 경영 튜토리얼 고정
  - 자원/선택/결과 확인 학습
  - 이상 징후는 노출하지 않거나 아주 약하게 둔다.
- 2일차 낮:
  - 1일차와 같은 구조의 약한 변주
  - 반복감은 주되 공포 장치로 설명하지 않는다.
- 3일차 시작쯤:
  - 최초 균열 발현 후보
  - 설명 대신 선택 구조의 어긋남으로 체감시킨다.

### 5. 검증 보강

추가 기능보다 재발 방지 검증을 먼저 보강한다.

- 표식 검증 유지:
  - `MARKS=100`
  - `stigma=50`
  - `brand=50`
  - unlock 조건 도달 가능
- 글리프 관련 검증 후보:
  - 제거안 채택 시 글리프 관련 파일/import/className이 남아 있으면 실패
  - 유지안 채택 시 `src/effects` 밖에 글리프 내부 구현이 있으면 실패
  - `index.html`이 `src/glyph-fall-fix.css`를 로드하면 실패
- 문서 검증 후보:
  - `200개`, `성흔100+낙인100`, `100+100` 같은 이전 장기 확장 문구가 현재 인계문에 남으면 실패
