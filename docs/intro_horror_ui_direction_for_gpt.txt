# 핑퐁 파일 정리 및 실제 수정 내역

사용자의 요청에 의해 이 파일의 이전 핑퐁 내용은 삭제했다.
이 문서는 이번 실제 코드 수정 내역만 남긴다.

## 수정 목적

낮의 빛무리와 밤의 붉은 글자비를 과도하게 방어적으로 분리하지 않고, 하나의 글리프 분위기 캔버스가 서식(format)에 따라 다른 표현을 그리도록 정리했다.

특수 상황에서 낮/밤 빛무리가 색상만 바뀌는 것이 아니라 밀도, 속도, 글자, 마스크, 흔들림 방식 등 다른 서식으로 변형될 수 있으므로, `day`/`night`만 보는 구조를 `format` 축으로 확장했다.

## 실제 수정 파일

- `src/App.jsx`
- `src/styles.css`
- `docs/intro_horror_ui_direction_for_gpt.txt`

## `src/App.jsx` 수정 내역

- `HorrorTextRainCanvas`를 `GlyphAtmosphereCanvas`로 변경했다.
- `textRain` director 항목을 `glyphAtmosphere`로 변경했다.
- `tone: "day" | "night"` 구조를 `format` 구조로 변경했다.
- 현재 기본 매핑은 다음과 같다.
  - 낮: `day-glow`
  - 밤: `night-red-rain`
- 추후 기본 밤 UI에 쓸 수 있도록 `night-blue-glow` 서식을 미리 열어두었다.
- 임계치 근접/오염 계열 확장을 위해 `critical-corruption` 서식도 열어두었다.
- 기존 `rainGlyphs`, `generateRain`, `HORROR_TEXT_RAIN_*` 이름을 글리프 분위기 계열 이름으로 바꾸었다.
- 캔버스 루프는 유지했다. DOM 글자 노드를 대량 생성하는 방식으로 바꾸지 않았다.

## `src/styles.css` 수정 내역

- `.horror-text-rain-canvas` 클래스를 `.glyph-atmosphere-canvas`로 변경했다.
- 기존 오버레이 위치, blend, reduced-motion 처리 흐름은 유지했다.

## 구조 판단

현재는 완전 분리보다 `GlyphAtmosphereCanvas` 하나에 `format`을 넘기는 방식이 더 적절하다고 판단했다.

이유:

- 낮 빛무리와 밤 붉은 글자비는 같은 캔버스 기반 글리프 입자 계열이다.
- 사용자는 특수 상황에서 같은 계열 효과가 다른 서식으로 변형될 수 있다고 했다.
- 따라서 낮/밤 컴포넌트를 무조건 분리하면 이후 특수 변형을 받기 불편할 수 있다.
- 반대로 눈동자, 붉은 문장 안개, static rows 같은 명백한 공포 전용 효과는 별도 컴포넌트로 유지하는 편이 맞다.

## 사용자 의견 추가

지나친 방어적 코딩보다 구현도에 신경쓸 것.

현재 단계에서는 가능한 모든 오독과 예외를 막는 방어 설계보다, 실제로 화면에 구현되고 테스트 가능한 단위 작업을 우선한다.
방어적 구조가 필요하더라도 구현 진도를 늦추거나, 코드가 과하게 우회적이 되어 기능 완성도를 떨어뜨리면 안 된다.

다만 이 의견은 위험한 대형 리팩터링, 저장 데이터 변경, 엔진 규칙 변경을 무조건 밀어붙이라는 뜻이 아니다.
기준은 다음에 가깝다.

- 작은 단위로 실제 구현한다.
- 사용자가 직접 확인할 수 있는 화면 변화를 만든다.
- 과한 추상화와 과한 예외 처리는 나중으로 미룬다.
- 구조 안전성은 유지하되, 기능 구현 자체를 방해하지 않는다.

## 현재 열린 서식 후보

- `day-glow`: 낮의 따뜻한 빛무리
- `night-blue-glow`: 평화로운 밤의 푸른 빛무리 후보
- `night-red-rain`: 기존 붉은 밤 글자비
- `critical-corruption`: 임계치 근접/오염 계열 후보

## 아직 하지 않은 것

- 기본 밤 배경을 푸른색으로 새로 만들지는 않았다.
- 기존 붉은 밤 배경을 `night-anomaly`로 실제 이동시키지는 않았다.
- `night-critical`의 구체 연출은 만들지 않았다.
- 밸런스, 공포 수치 표시 조건, 빨간 선택지 수량은 바꾸지 않았다.

## 다음 구현 후보

1. UI 프리셋 director를 별도로 만든다.
   - `timePhase`: `day` / `night`
   - `uiStage`: `calm` / `anomaly` / `critical`
   - `glyphFormat`: `day-glow` / `night-blue-glow` / `night-red-rain` / `critical-corruption` / 특수 서식

2. 기본 밤 `night-calm`을 만든다.
   - 푸르스름한 배경
   - `night-blue-glow` 글리프 분위기
   - 공포 UI 없음

3. 기존 붉은 밤은 `night-anomaly`로 이동한다.

4. 특수 이벤트/성흔/기록/임계치 상태에서 `glyphFormat`을 바꾸는 규칙을 추가한다.
