# 도입부 공포 UI 방향 메모

목적: 다른 GPT/코덱스가 초반 UI와 선택지 색상 방향을 착각하지 않도록 공유하는 짧은 설계 메모.

## 핵심 의견

도입부에서 빨간 선택지가 너무 많으면 유저가 위험 신호를 배우기 전에 혼란을 느낄 가능성이 크다.
빨간색은 일반 선택지 색이 아니라, 금기/위험/돌이킬 수 없음/제물/피와 연결된 특별 신호로 아껴 써야 한다.

초반은 겉으로는 평화로운 영지 관리처럼 진행하고, 공포는 문장과 상황의 위화감으로 가볍게 암시하는 편이 좋다.
유저가 먼저 정상적인 낮의 흐름을 익힌 뒤, 그 정상성이 조금씩 무너질 때 공포 효과가 커진다.

## 초반 낮 방향

- 낮에는 자원, 사람, 영지 관리가 중심이다.
- 공포도, 이상현상, 침식 같은 수치는 기본적으로 숨긴다.
- 내부 계산은 유지하되, 유저에게는 숫자 대신 분위기 문장으로만 암시한다.
- 예: "저택은 조용하다", "창고 쪽 공기가 눅눅하다", "하인들이 특정 방을 피한다".
- 선택지는 대부분 평범한 색을 사용한다.
- 불길한 선택지도 초반에는 문구로만 암시하고, 색은 일반 선택지와 크게 다르게 만들지 않는다.

## 빨간 선택지 규칙

- 도입부와 초반 낮에는 빨간 선택지를 0~1개 수준으로 제한한다.
- 빨간 선택지는 정말 특별한 선택에만 사용한다.
- 사용 예: 피, 제물, 금기, 돌이킬 수 없는 선택, 명백한 위험, 밤/지하와 직접 연결된 선택.
- 단순히 수상하거나 분위기가 어두운 선택에는 빨간색을 쓰지 않는다.
- 빨간색이 많아지면 위험 신호가 아니라 기본 UI처럼 보이므로 효과가 약해진다.

## 공포 수치 표시 규칙

- 낮 기본 상태: 공포/이상현상 수치 표시 안 함.
- 낮 이상 징후 낮음: 짧은 분위기 문장만 표시.
- 낮 이상 징후 중간: UI 일부가 흐리게 반응하거나 텍스트가 불안정해짐.
- 낮 이상 징후 높음: 공포/이상현상 패널이 제한적으로 켜짐.
- 밤, 지하, 특정 이벤트, 임계치 초과, 진실 발견 이후에는 수치 또는 위험 표시를 활성화할 수 있다.

## 진행 리듬

초반 낮: 평화로운 관리 + 약한 암시.
초반 밤: 처음으로 빨간 선택지와 공포 UI의 의미를 보여줌.
중반 이후: 공포/이상현상이 낮 화면에도 침투하기 시작.
후반: 낮/밤 경계가 약해지고, 숨겨진 수치가 UI에 직접 영향을 주기 시작.

## 구현 시 주의

- 공포 수치를 숨기는 것은 계산을 멈추는 뜻이 아니다. 내부 상태는 계속 누적된다.
- 유저가 확률이나 위험을 판단해야 하는 시점에는 충분한 신호를 줘야 한다.
- 다만 초반에는 숫자보다 문장, 색보다 분위기, 노골적 경고보다 위화감을 우선한다.
- 빨간색은 시스템 언어로 보존한다. 많이 쓰면 의미가 사라진다.

## 결론

도입부는 평화롭게 보이되 완전히 안전하게 보이면 안 된다.
초반의 목표는 공포를 설명하는 것이 아니라, 나중에 망가질 정상 상태를 유저에게 먼저 체감시키는 것이다.
