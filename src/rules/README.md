# 로젠탈 영지 관리일지 규칙

## 현재 구현 범위

- 본편 시작 이전 로젠탈 영지의 1~7일차와 축별 8일차 첫 장면
- 낮 5턴: 자원 채집, NPC 상호작용, 조사, 수련, 휴식, 기타 중 하루 다섯 행동군 선택
- 밤 탐사: 동행자 선택, 진입 방향 선택, 탐사 3~10단계, 최종 퍼즐 또는 전투
- 60개 성인·성녀·천사 시드와 12개 3단계 특수 사건군
- 낮에서 밤으로 넘어갈 때 해가 떨어지는 전환을 표시한다. 밤이 끝나면 별도 전환 없이 다음 날로 넘어간다.
- 모든 내레이션과 인물 대사를 동일한 대화형 UI로 표시하고, 대화 뒤에 선택지 또는 상태 변화 화면을 연다.
- 대화형 UI의 문단별 진행, 첫 대화까지 돌아가는 이전 버튼, 우측 상단 스킵
- 시드 공개 팝업과 화자별 대화 표시
- 규칙 탭, 튜토리얼 종료 안내, 패시브 3개와 단일 MARKS 표식 장착
- 자동 저장 1개와 수동 저장 10개

## 제물 규칙

- 제물 카운트는 1일차부터 7일차 밤까지만 열린다.
- 인간 동행자의 사망과 영구 실종은 즉시 제물로 계산한다.
- 변질은 그 자체로 제물이 아니다. 변질된 인간을 플레이어가 처치한 순간 제물로 계산하고 유품을 얻는다.
- 밤 탐사에서 동행자가 변질되면 귀환하기 전에 살려둘지 처치할지 반드시 결정한다.
- 유품 전달은 낮의 NPC 상호작용을 사용하며 제물 카운트를 다시 올리지 않는다.
- 같은 인간은 한 번만 제물로 계산한다.
- 사망, 실종, 변질된 동행자에게 동일한 영구 상태를 다시 적용하지 않는다.
- 8일차 이후의 인간 손실은 제물 분기 조건에 포함하지 않는다.

## 분기

- 제물 3명을 채우면 8일차 변질 축이 우선 열린다.
- 채우지 않으면 로젠탈 주민들이 처음 경험하는 정상 축의 여덟째 날이 열린다.
- 지하 진입 포기 7회는 `정식 영주` 제물 사망 엔딩으로 이어진다.
- 낮 포기와 탐사 중 귀환은 일반 포기다. 첫 포기부터 큰 축의 엔딩이 닫히고 세 번째 포기 뒤 다음 아침에 사망한다.

## 탐사와 회복

- 일반 탐사의 기본 스태미나 변화는 성공 시 `-1`, 실패 시 `-2`다.
- `동행자와 길을 확보하며 지나간다`의 기본 성공률은 100%이며 결단 저하 등 성공률 감소 규칙을 적용한다.
- 동행자가 생존 상태가 아니면 동행자를 사용하는 탐사 선택지를 고를 수 없다.
- 스태미나가 음수가 되면 결과 확인 뒤 낮에는 밤으로, 밤에는 다음 날로 강제 귀환한다.
- 휴식은 체력과 스태미나를 각각 1~5 회복하며 생명 성향에 따라 회복량이 달라진다.
- 모든 일반 지하 탐사 선택지에는 `휴식한다`가 함께 표시된다. 체력 `+1`, 스태미나 `+3`, 결단 `-1`, 통찰 `-1`, 공포 `+3`, 이상 징후 `+5`를 적용하고 다음 탐사 단계로 진행한다.
- 통찰이 음수면 선택 결과의 양수 변화만 같은 크기의 음수로 뒤집는다. 원래 음수인 변화는 그대로 적용한다.
- 다음 날이 시작될 때 음수 통찰은 `0`으로 복구된다.

## 표시 규칙

- 자원 상태 문구는 현재 단계와 위험도에 따라 글자 색상이 달라진다. 공포는 낮을수록 좋은 색상으로 표시한다.
- 모든 선택지는 마우스를 올리면 행동 목적과 주요 이득·손실을 설명한다.
- 선택 결과의 변화는 주인공 능력치, 영지 자원, 영지 상태, 성향으로 나누어 표시한다.

## 시작 시드

- 새 게임 버튼을 누른 순간의 로컬 시간 초 `00~59`가 특수 시드를 결정한다.
- 시드는 `성 미카엘 대천사의 달` 형식으로 시작 후에만 공개한다.
- 시드 특성은 특정 행동·위험·성공·실패·결과 변화에 반응하는 조건부 혼합 가중치로 표시한다.
- 각 시드는 서로 다른 이득 조건 하나와 손실 조건 하나를 가지며, 모든 가중치는 `x0.9~x1.1` 범위다.
- 실제 계산은 소수점 한 자리로 반올림하고, 화면에는 반올림 전 값을 소수점 한 자리에서 버려 표시한다.
- 인접한 다섯 시드는 하나의 특수 사건군을 공유한다.
- 특수 사건은 해당 날짜의 첫 낮 행동 뒤에 등장한다.
- 일반 후보와 판정은 별도 내부 난수로 고정한다.

## 데이터와 엔진

- `src/data/saintSeeds.js`: 60개 시드 이름, 상징, 혼합 규칙, 사건군 배정
- `src/data/rosenthal/*.js`: 낮 행동, NPC/동행자, 탐사 사건, 최종 사건, 프롤로그/밤 진입 스크립트 수정 원본
- `src/data/rosenthalScriptContent.js`: Rosenthal 스크립트 데이터 aggregate wrapper
- `src/data/rosenthalContent.js`: Rosenthal 런타임 import 호환용 wrapper
- `src/data/tutorial/*.js`: 튜토리얼 화면/구형 낮·밤 진행/엔딩·포기 결과 수정 원본
- `src/data/tutorialContent.js`: 튜토리얼 런타임 import 호환용 aggregate wrapper
- `src/engine/rosenthalEngine.js`: 상태 전이, 판정, 제물, 분기, 포기, 유품 처리
- `src/engine/saveManager.js`: 자동 저장과 수동 저장 슬롯
- 기존 `systemRules.js`와 `rulesEngine.js`: 성향, 직업, 칭호, 도감/휴대/장착 표식, 패시브 및 효과 적용 순서

효과 적용 순서는 성향, 직업, 칭호, 휴대 표식, 장착 표식, 패시브 1~3을 유지한다. 성인의 달 조건부 특성은 그 뒤, 음수 통찰 반전 전에 적용한다.

## 사건군 확장 승인

새 사건군을 추가하기 전에는 다음 내용을 먼저 승인받는다.

1. 사건군의 전제, 발생 기간, 장소
2. 영향을 받는 NPC와 가능한 사망, 실종, 변질 결과
3. 분기, 엔딩, 본편 인물 및 정본 규칙에 미치는 영향

승인된 사건군 안의 일반 선택지 문장, 비영구 결과, 기본 수치는 별도 승인 없이 작성한다.

## 정본 경계

- 게임의 연금술사는 본편 시작 이전의 동일 인물이다.
- 첫 변질 회차에서 세레스가 플레이어를 처형하고 아벨이 로젠탈을 닫는 결말만 본편 과거의 정본이다.
- 다른 회차와 엔딩은 플레이 가능한 비정본 가능성이다.
- 이번 구현은 작품 정본 문서를 수정하지 않는다.

신규 저장은 `localStorage`의 `rosenthal-estate-save-v1`을 사용한다. 기존 `eldroa-estate-run-v9` 저장은 삭제하지 않는다.

## 표식 구조 (확장용)

- 표식은 `성향`(affinity)과 `단독`(standalone)으로 분리되어 관리한다.
- 성향 기반 표식은 `AFFINITY_MARK_GROUPS`에 두며, `life~suspicion` 10성향을 유지한다.
- 성향 성흔/낙인은 기존 성향별 기본 3개씩(`총 60개`)과 세 번째 `capstone` 구조를 보존한다.
- 현재 성향형 표식은 `총 94개`이며, `life~exorcism`은 성흔 5개/낙인 5개, `execution~suspicion`은 성흔 4개/낙인 4개를 가진다.
- 단독 표식은 `STANDALONE_MARKS`로 별도 관리하며, 현재는 샘플 6개만 반영한다.
- 현재 표식 총합 목표는 성흔/낙인 도합 `100개`다. 이전 장기 확장안은 현재 기준이 아니다.
- 분기 해방 조건은 현재 카탈로그 최대치인 성흔 50개/낙인 50개/전체 100개 안에서 모두 도달 가능해야 한다.
