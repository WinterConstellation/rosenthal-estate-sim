# 규칙 데이터 관리

튜토리얼은 도입부 가이드의 단일 구조를 사용한다.

1. 오전, 오후, 저녁에 평범한 영지 업무를 하나씩 선택한다.
2. 각 행동 뒤에는 다음 시간대로 넘어가기 전 내레이션이 출력된다.
3. 선택한 업무가 성향을 누적하고 밤에 부를 수 있는 사람 또는 물건을 연다.
4. 밤에는 저택 지하로 함께 내려갈 대상을 고르거나 진입을 포기한다.
5. 밤이 끝나면 다음 날로 넘어가며 낮 행동 후보가 다시 구성된다.
6. 낮과 밤의 선택 기록으로 초기 직업, 칭호, 접두·접미 성흔을 정한다.

## 화면 역할

- 대사와 내레이션은 화면 하단 전용 창에 표시한다.
- 선택 평가, 영지 상태, 날짜 변경은 중앙 상태 창에 표시한다.
- 엔딩은 다른 창을 남기지 않는 독립 전체 화면으로 표시한다.

## 구현된 엔딩

`정식 영주`:

- 7일차 밤까지 진행한다.
- 모든 밤에 지하 진입을 포기한다.
- 선택 결과 적용 전후를 포함해 어떤 자원, 영지 상태, 성향도 한 번도 0 아래로 내려가지 않는다.

## 파일 역할

- `systemRules.js`: 성향, 직업, 칭호, 성흔, 패시브와 실행 시드의 숨은 규칙
- `tutorialRules.js`: 프롤로그, 낮 행동, 밤 선택, 잡부 이름 후속 선택, 포기 결과
- `src/engine/seed.js`: 같은 시드에서 같은 후보 순서를 만드는 함수
- `src/engine/rulesEngine.js`: 후보 가중치 계산, 선택 결과 적용, 직업·칭호·성흔 결정

## 튜토리얼 선택지 원칙

- 선택지 버튼에는 행동 문장만 표시한다.
- 선택 전에 성향 증가량, 결과, 정답 여부를 설명하지 않는다.
- 중립 선택은 흰색, 위험한 선택은 붉은색에서 검붉은색으로 표시한다.
- 결과의 선악을 판단할 수 없고 영향이 극단적일 수 있는 선택은 금색과 검정으로 표시한다.
- 낮 후보는 아직 선택하지 않은 행동 중 시드와 현재 성향에 따라 3~5개를 보여준다.
- 밤 후보는 낮에 접촉한 사람과 물건을 우선하며, 조건을 만족하지 못한 선택지도 잠긴 상태로 보일 수 있다.
- 모든 낮·밤 선택 화면에는 `포기한다`가 별도로 존재한다.
- 아무 변화도 남기지 못한 선택은 손실과 같은 평가를 받는다.

## 도입부에서 제외하는 것

- 대성당
- 본편 인물의 직접 등장 또는 1:1 대응
- 별도 사건군과 사건군별 메인 흐름
- 세계 규칙의 정답을 설명하는 대사

## 효과 적용 순서

1. 성향
2. 직업
3. 칭호
4. 접두 성흔
5. 접미 성흔
6. 패시브 1
7. 패시브 2
8. 패시브 3

## 주인공 능력치와 음수 규칙

- 선택 결과에는 실제 적용된 능력치, 자원, 영지 상태, 성향, 호감도 변화를 표시한다.
- 변화량 절댓값이 `1~2`면 화살표 1개, `3~5`면 2개, `6 이상`이면 3개로 표시한다.
- `stats`는 선택으로 직접 조정되는 주인공 능력치 데이터다.
- 체력이 음수가 되면 결과 확인 뒤 게임이 종료된다.
- 통찰이 음수면 다음 선택에서 모든 양수 획득값이 같은 크기의 음수로 반전된다.
- 결단이 음수면 `successChance`가 있는 선택의 성공 확률이 절반이 된다.
- 매력이 음수면 양수 호감도가 있는 `npcId` 관련 선택 시 해당 호감도가 1 감소한다.
- 신앙이 음수면 `divineBonus` 선택을 고를 수 없다. `heresy`이거나 신앙을 낮추는 선택은 고를 수 있다.
- 스태미나가 음수가 되면 결과 확인 뒤 낮은 밤으로, 밤은 다음 날로 강제 전환된다.
- 낮에서 밤으로 넘어가거나 다음 날이 시작되면 스태미나는 10으로 회복된다.

확률 선택은 `successChance`와 선택적인 `failure` 결과를 사용한다. 인물 관련 선택은 `npcId`, 신성 보너스 선택은 `divineBonus`로 명시한다.

진행 상황은 `localStorage`의 `eldroa-estate-run-v9`에 저장한다.
